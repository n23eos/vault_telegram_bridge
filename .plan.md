# План релиза 0.3.0 — ветка `dev`

## Цель

Сделать Vault Telegram Bridge полезнее для широкой аудитории: сообщения можно
раскладывать по тематическим заметкам, голосовые — превращать в текст через
выбранный пользователем OpenAI-совместимый сервис, а интерфейс — использовать
на русском языке. Подготовить репозиторий к публикации версии 0.3.0 в каталоге
Community Plugins.

## Пользовательские сценарии

1. Пользователь отправляет боту `#idea Купить проектор`; запись попадает в
   `Inbox/Ideas.md`, а служебный тег `#idea` не засоряет текст.
2. Пользователь задаёт маршрут с датой (`Topics/idea/YYYY-MM.md`) и получает
   тематические месячные заметки. Если ни один маршрут не совпал, сохраняется
   текущее поведение с daily note.
3. Пользователь включает транскрипцию, задаёт URL, API key и модель; voice,
   audio и video note сохраняются как вложения и получают расшифровку. Ошибка
   STT не блокирует синхронизацию и не теряет вложение.
4. Пользователь с русским интерфейсом Obsidian видит настройки и уведомления на
   русском; неизвестная локаль продолжает получать английский интерфейс.

## Инварианты и риски

- Cursor двигается только после записи всех сообщений; ошибка STT сама по себе
  не должна повторно доставлять уже сохранённое вложение.
- Первый маршрут в порядке настроек, тег которого присутствует в сообщении,
  побеждает. Сравнение регистронезависимое, ведущий `#` необязателен.
- API key хранится в `data.json` так же, как bot token, и никогда не логируется.
- STT выключен по умолчанию; сетевой вызов выполняется только для voice/audio/
  video note и только после успешного получения байтов.
- Пути маршрутов vault-relative; traversal и недопустимые символы отклоняются.
- Реальный Telegram/STT smoke-test требует пользовательских секретов и остаётся
  ручным шагом перед тегом релиза.
